公司檔案 Company Profile
路徑： /profile | 權限： ADMIN, PM | 所屬旅程： I. 系統初始化與組織設定

1. 模組定位 (Core Purpose)
提供企業核心資訊、ESG 策略目標與組織成員管理，是 ESG GO 系統運作的基礎設定中心，確保所有報告數據的基礎一致性與合規性。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 永續專案經理 (PM) 或系統管理員 (ADMIN)。他們需要一個集中且易於管理的介面來設定企業的基本資料、ESG 策略目標與管理成員權限，以確保所有報告數據的基礎一致性與合規性。過去這些資訊可能散落在多個文件或系統中，更新困難且容易出錯。
體驗高光時刻 (Aha Moment)： 當 PM 在「ESG 策略目標」區塊更新了碳排放減量目標，並看到 Dashboard 的預測曲線即時響應，感受到「所有設定都緊密連結，我的變更立即生效」的掌控感與效率。
操作軌跡：
1. 登入系統後，導航至左側選單的「公司檔案」。
2. 點擊「基本資料」頁籤，更新公司名稱、統一編號、產業別與聯絡資訊。
3. 切換至「ESG 策略目標」頁籤，設定本年度的碳排放減量百分比、再生能源使用比例等目標。
4. 進入「成員管理」頁籤，新增或調整組織成員的角色與權限。
5. 點擊「儲存」按鈕，確認所有變更已生效並反映在相關 Dashboard 上。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 採用「頂部 Tab 導航」搭配「單欄式表單佈局」。每個 Tab 代表一個設定區塊（如：基本資料、ESG 目標、成員管理），內容區塊則以卡片形式呈現各項設定表單。
核心液態玻璃元件： BrandTab (頁籤導航)、BrandInput (通用輸入框)、BrandSelect (下拉選單)、BrandAvatarUploader (頭像上傳元件)、BrandStatusBadge (成員狀態標籤)、BrandToggleSwitch (功能開關)。
行動端適配 (RWD)： < 768px 時，頂部 Tab 導航自動收合為「下拉選單 (Dropdown Menu)」，所有表單元素必須垂直堆疊，確保輸入框與按鈕的點擊區域足夠。成員列表則轉化為「堆疊式卡片」，每張卡片顯示一位成員的關鍵資訊。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向： 使用者在表單中輸入或修改的資料，透過 API 寫入 `company_profiles` (公司基本資料)、`esg_targets` (ESG 策略目標) 及 `user_roles` (成員權限) 等資料表。這些變更會即時影響 Dashboard 上的相關 KPI 顯示與報告生成邏輯。
5T 實踐點：
[T1 Tangible 具體]：將抽象的「ESG 策略」具體化為可量化的數值目標（如：碳排放量減少 15%），並在介面中以進度條或圖表形式呈現，讓管理者能直觀感受目標進度。
[T2 Traceable 追溯]：所有對公司基本資料、ESG 目標及成員權限的修改，都會在後台生成詳細的 Audit Log，記錄「誰」、「在何時」、「修改了什麼」以及「修改前後的值」，確保所有變更可追溯。
[T3 Trackable 追蹤]：系統會追蹤 ESG 目標的達成進度，並在 Dashboard 上提供歷史趨勢圖，讓管理者能持續追蹤策略成效，並在目標偏離時發出預警。
[T5 Trustworthy 信任]：成員權限管理採用 RBAC (Role-Based Access Control) 模型，確保只有具備相應權限的使用者才能修改關鍵設定，防止未經授權的資料變更，維護系統的安全性與資料完整性。

5. 功能項目解說和使用技術 (Features & Tech Stack)
公司基本資料編輯： 允許管理者修改公司名稱、統一編號、產業別、聯絡資訊等。技術使用 React Hook Form 進行表單狀態管理與驗證，資料儲存於 Supabase `company_profiles` 表。
ESG 策略目標設定： 提供輸入框與選擇器，設定年度碳排放減量、再生能源使用比例等目標。技術使用 Chart.js 實時預覽目標達成曲線，並透過 Zustand 進行跨元件狀態同步。
組織成員與權限管理： 顯示所有成員列表，可新增、編輯或刪除成員，並分配預設角色（ADMIN, PM, USER）。技術使用 Supabase Auth 進行使用者身份驗證與管理，並實作客製化的 RBAC (Role-Based Access Control) 邏輯。
公司 Logo 與憑證上傳： 支援拖曳上傳公司 Logo 或相關認證憑證。技術使用 react-dropzone 處理檔案拖曳事件，並將檔案上傳至 Supabase Storage，同時進行檔案類型與大小驗證。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 當公司名稱或地址輸入過長時，輸入框內的文字必須自動截斷並顯示省略號 (`...`)，並在 Tooltip 中顯示完整內容，不可撐破表單佈局。成員列表中的角色標籤在行動端必須正確換行或縮小字體，不可溢出卡片邊界。
🚨 邏輯/體驗紅線： 當使用者嘗試儲存包含無效資料（如：非數字的目標百分比、格式錯誤的電子郵件）的表單時，必須在對應欄位下方顯示清晰的錯誤提示，並阻止儲存操作。刪除成員時，必須彈出二次確認對話框，並檢查該成員是否仍有未完成的任務或報告草稿，若有則提示管理者進行任務轉移，避免資料孤島。
