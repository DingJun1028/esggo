閱讀室 Reading Room
路徑： /reading-room | 權限： PM, ANALYST, AUDITOR | 所屬旅程： I. 知識探索與學習

1. 模組定位 (Core Purpose)
提供一站式 ESG 知識庫，彙整全球永續報告標準、法規指引、產業最佳實踐與內部知識文件，幫助使用者快速學習、理解並應用 ESG 資訊。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 永續分析師 (Analyst) 或專案經理 (PM)。他們經常在浩瀚的 ESG 資訊海中迷失，難以快速找到特定 GRI/ISSB 條款的解釋、最新法規更新或產業標竿案例，導致報告撰寫效率低落且合規風險增高。
體驗高光時刻 (Aha Moment)： 當分析師輸入「GRI 305 排放量計算」時，系統不僅精準呈現 GRI 官方指引，還自動推薦「本公司去年報告中 305 章節的撰寫範例」與「同產業標竿企業的公開報告連結」。分析師瞬間感受到「這就是我需要的，而且還幫我整理好了！」的效率提升。
操作軌跡：
1. 進入閱讀室，在頂部搜尋框輸入關鍵字「ISSB S2」。
2. 瀏覽搜尋結果，點擊「ISSB S2 實施指南」文件。
3. 閱讀文件內容，並利用內建的「重點標註」功能標記重要段落。
4. 將該文件加入「我的收藏」，以便日後快速查閱。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 採用「左側導航篩選 + 中央內容展示」的雙欄佈局。左側為「知識分類樹」與「多維度篩選器」，中央為「文件列表」與「內容預覽區」。頂部設有「全域搜尋列」。
核心液態玻璃元件： BrandSearchBar (具備即時建議)、FilterTagGroup (可多選)、DocCard (文件預覽卡片，顯示標題、類型、更新日期)、ContentViewer (支援 PDF/Markdown/HTML 渲染)。
行動端適配 (RWD)： < 768px 時，左側導航篩選器自動收合為「底部 Tab」或「漢堡選單」，中央文件列表轉化為「垂直堆疊卡片」，內容預覽區則自動滿版，確保閱讀體驗。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向： 文件內容與元數據 (標題、分類、標籤、來源) 儲存於 `knowledge_base_docs` 資料表。搜尋索引由 ElasticSearch 或 Supabase Full-Text Search 建立。使用者行為 (收藏、標註) 則寫入 `user_reading_logs`。
5T 實踐點：
[T4 Transparent 透明]： 每一份文件都明確標示其「來源類型」（如：GRI 官方、ISSB 官方、內部政策、產業研究報告）與「發佈日期」，確保資訊出處清晰可見。
[T5 Trustworthy 信任]： 針對官方標準文件，系統會定期比對其 Hash 值與官方最新版本，若有更新或內容異動，會自動標示「已過期」或「待更新」警示。
[T3 Trackable 追蹤]： 內部知識文件支援版本控制，每次編輯或更新都會留下編輯者與時間戳記，方便追溯內容演變。

5. 功能項目解說和使用技術 (Features & Tech Stack)
全域智慧搜尋與篩選： 支援關鍵字、文件類型、發佈日期、GRI/ISSB 指標等多維度組合篩選。技術使用 ElasticSearch 或 Supabase Full-Text Search 進行索引與查詢，前端使用 React Hook Form 處理複雜表單邏輯。
標準文件嵌入與交叉引用： 能夠直接在頁面內預覽 PDF 或 Markdown 格式的標準文件，並支援文件內部的關鍵字高亮與章節跳轉。技術使用 React-PDF 或 Markdown-to-HTML 渲染器，並結合自定義的錨點連結解析器。
個人化收藏與標註： 使用者可將常用文件加入「我的收藏」，並在文件內容中進行文字標註與筆記。技術使用 Supabase 的 `user_preferences` 表儲存收藏列表，並利用 Quill.js 或 TipTap 編輯器實現內容標註功能。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 當文件標題過長時，在文件列表中必須自動截斷並顯示省略號 (text-overflow: ellipsis)，不可撐開卡片或表格欄位。
🚨 UI 跑版紅線： 在行動端，內容預覽區的圖片或表格必須自動縮放以適應螢幕寬度，不可出現橫向滾動條或內容溢出。
🚨 邏輯/體驗紅線： 搜尋結果必須在 2 秒內返回，且相關性排序必須符合預期。若搜尋無結果，應提供「建議關鍵字」或「聯絡管理員」的友善提示，而非空白頁面。
🚨 邏輯/體驗紅線： 任何外部連結 (如 GRI 官方網站) 必須定期檢查其有效性，若連結失效，應自動標示為「無效連結」並通知管理員，不可導向 404 頁面。
