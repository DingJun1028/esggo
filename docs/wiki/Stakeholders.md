# 利害關係人 Stakeholders [Stakeholders]
路徑： /stakeholders | 權限： PM, CSO | 所屬旅程： II. 策略盤點與分派

1. 模組定位 (Core Purpose)
利害關係人模組提供一站式平台，透過影響力矩陣、情感追蹤與互動日誌，系統化管理各利害關係群體，以精準識別關鍵議題並優化溝通策略。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 永續專案經理 (PM)、永續長 (CSO) 或溝通部門主管。他們通常面對來自投資人、員工、客戶等多方利害關係人的碎片化回饋，難以有效整合、追蹤互動歷史，也無法直觀判斷哪些群體或議題需要優先回應，導致溝通效率低下且策略制定缺乏依據。
體驗高光時刻 (Aha Moment)： 當 PM 在影響力矩陣上調整某群體的「關注度」滑桿時，該群體的氣泡立刻動態飛入「高影響力/高關注度」象限，同時右側的「待辦溝通清單」自動將該群體相關議題置頂。PM 瞬間感受到「策略優先級一目瞭然」的掌控感與效率提升。
操作軌跡：
1. 進入利害關係人頁面，檢視現有利害關係人群體總覽。
2. 點擊「新增群體」，填寫基本資料並設定其初始「影響力」與「關注度」分數。
3. 瀏覽動態生成的「影響力-關注度矩陣」，識別出位於「核心重大」象限的群體。
4. 點擊特定群體，進入其詳細頁面，記錄最新的互動內容與情感變化。
5. 根據矩陣分析結果，調整溝通策略並分配資源。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 採用「頂部篩選與總覽卡片區」+「中央互動式影響力矩陣圖表」+「下方分頁式互動日誌與關注議題列表」的三段式佈局。
核心液態玻璃元件： `StakeholderMatrixChart` (支援拖曳與即時更新的氣泡圖)、`SentimentBadge` (顯示情感趨勢的微型圖表與標籤)、`InteractionLogCard` (具備時間軸與標籤的互動紀錄卡片)。
行動端適配 (RWD)： < 768px 時，頂部篩選區收合至「底部彈出式選單 (Bottom Sheet)」。中央的「影響力矩陣」自動轉化為「優先級列表 (Priority List View)」，僅顯示關鍵群體與其分數，並提供「點擊展開全螢幕圖表」選項。下方的「互動日誌」與「關注議題」則以堆疊式卡片呈現，確保內容可讀性。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向： 使用者輸入的利害關係人群體資訊、影響力/關注度分數、互動日誌與情感標籤，會即時寫入 `stakeholders`、`stakeholder_interactions` 和 `stakeholder_issues` 等 Supabase 資料表。這些數據將直接影響「重大性矩陣」的議題優先級排序，並作為「永續報告」中利害關係人溝通章節的數據來源。
5T 實踐點：
[T1 Tangible 具體]： 將抽象的「影響力」與「關注度」評分（1-5分）具體化為二維座標軸上的氣泡位置與大小，讓使用者能直觀理解各群體的相對重要性。
[T2 Traceable 追溯]： 每次利害關係人群體的「影響力」或「關注度」分數被修改時，系統會自動在該群體的 Audit Log 中記錄「Score_Updated」事件，包含修改者、時間與新舊數值。
[T3 Trackable 追蹤]： 互動日誌中的情感標籤（正面/中立/負面）會被系統追蹤，並在群體詳細頁面生成情感趨勢圖，幫助使用者長期監測利害關係人的情緒變化。
[T4 Transparent 透明]： 影響力矩陣的計算邏輯（例如：如何將分數映射到座標軸）會在頁面底部提供一個可展開的「計算說明」區塊，確保使用者理解數據呈現的依據。
[T5 Trustworthy 信任]： 只有具備 PM 或 CSO 權限的角色才能修改利害關係人的核心分數與刪除互動紀錄，確保數據的權威性與不可篡改性。

5. 功能項目解說和使用技術 (Features & Tech Stack)
影響力矩陣互動圖表： 支援氣泡拖曳、即時分數更新與 Tooltip 懸浮提示。技術使用 D3.js 進行客製化圖表渲染，結合 React Context API 進行狀態管理，確保圖表與分數滑桿的毫秒級同步。
情感追蹤與分析： 透過自然語言處理 (NLP) 技術，對互動日誌中的文字內容進行情感傾向分析，並標註為正面、中立或負面。技術使用 Google Cloud Natural Language API (或自建基於 NLTK 的 Python 微服務) 進行後端處理，前端則以 `SentimentBadge` 元件呈現結果。
互動日誌管理： 提供富文本編輯器，支援文字格式、圖片上傳與文件附件。技術使用 TipTap 或 Lexical 編輯器核心，搭配 Supabase Storage 進行文件儲存。
群體關注議題關聯： 允許將特定 ESG 議題與利害關係人群體進行多對多關聯，並支援關鍵字搜尋與篩選。技術使用 Algolia 或 Elasticsearch 進行高效搜尋與過濾，前端使用 React Hook Form 處理表單與篩選邏輯。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 當利害關係人群體名稱過長時，矩陣氣泡上的文字必須觸發 `text-overflow: ellipsis` 並在 Tooltip 中顯示完整名稱，絕對不可撐破氣泡或導致圖表佈局混亂。
🚨 邏輯/體驗紅線： 當使用者嘗試刪除一個已與「重大性矩陣」或「永續報告」模組深度關聯的利害關係人群體時，系統必須彈出二次確認對話框，並明確提示刪除可能帶來的影響，而非直接執行刪除。

7. 矩陣關聯 (Matrix Connection)
上游數據： 來自企業溝通部門、投資人關係 (IR) 管道。
下游影響： 直接提供加權數據給 `/materiality` 進行議題評估。
依賴組件： StakeholderMatrixChart, SentimentBadge, InteractionLogCard.
