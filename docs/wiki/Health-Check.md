# ESG 健檢儀表板 ESG Health Check Dashboard [Health Check]
路徑： /health-check | 權限： PM, CSO, ADMIN | 所屬旅程： III. 數據採集與分析

1. 模組定位 (Core Purpose)
ESG 健檢儀表板提供企業快速、全面的 ESG 表現評估，透過數據驅動的洞察與 AI 賦能的改善建議，協助企業識別潛在風險、發掘機會點，並制定符合國際標準的永續發展策略。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 永續專案經理 (PM) 或永續長 (CSO)。他們面臨的痛點是缺乏一套標準化且自動化的工具來評估企業在各 ESG 議題上的表現，難以量化進度，也無法有效向高層匯報或制定具體改善計畫。手動收集數據、交叉比對標準、撰寫評估報告的過程耗時且容易出錯。
體驗高光時刻 (Aha Moment)： 當使用者完成數據輸入後，儀表板立即以直觀的雷達圖或熱力圖呈現企業在 15 項核心 ESG 議題上的得分與行業平均對比，並自動彈出「前三大改善建議」卡片，讓使用者瞬間感受到「原來我的企業表現一目瞭然，而且有明確的下一步行動！」的清晰與效率。
操作軌跡：
1. 進入「ESG 健檢儀表板」，系統自動載入最新一期的企業數據。
2. 檢閱儀表板上的總體得分、各議題細項得分與趨勢分析。
3. 點擊「生成改善計畫」，系統根據評估結果與內建知識庫，自動生成一份包含 90 天里程碑的行動計畫草稿。
4. 針對特定低分議題，點擊「AI 洞察」按鈕，獲取更深入的風險分析與最佳實踐案例。
5. 將最終確認的行動計畫分派給相關部門，並設定追蹤提醒。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 採用「多卡片儀表板」佈局。頂部為總體得分與趨勢概覽卡片，下方分為左右兩欄，左側為「ESG 議題雷達圖/熱力圖」與「議題列表」，右側為「AI 改善建議」與「90 天行動計畫進度」卡片。
核心液態玻璃元件： BrandScoreCard (總體得分卡片，帶有動態數字滾動效果)、RadarChart (互動式雷達圖，可切換不同維度)、BrandProgressGauge (90 天計畫進度環形圖)、AIInsightCard (AI 建議卡片，帶有漸變背景)。
行動端適配 (RWD)： < 768px 時，所有卡片自動堆疊為單欄佈局。雷達圖/熱力圖預設隱藏，僅顯示總體得分與議題列表。點擊議題列表中的項目，可展開查看詳細得分與 AI 建議，並提供「全螢幕圖表模式」供用戶手動切換。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向： 系統從 `esg_data_warehouse` 撈取企業各項 ESG 數據，經由 `health_check_engine` 進行評分運算，結果儲存至 `esg_health_scores` 資料表。AI 建議則基於 `esg_knowledge_base` 與 `industry_benchmarks` 生成，並與 `action_plans` 資料表關聯。評分結果會影響 `esg_dashboard_kpis` 中的「整體 ESG 表現」指標。
5T 實踐點：
[T1 Tangible 具體]： 將抽象的 ESG 議題（如「生物多樣性」）透過標準化的評分模型，具體化為 0-100 的分數，並以雷達圖、熱力圖等視覺化方式呈現，讓企業表現一目瞭然。
[T2 Traceable 追溯]： 每個議題的得分都可點擊鑽取 (Drill-down)，顯示其計算公式、引用數據來源（連結至證據金庫 Vault 中的具體文件），以及評分模型版本，確保評分過程可追溯。
[T3 Trackable 追蹤]： 90 天改善計畫中的每個行動項目都設有進度條與負責人，系統會自動追蹤其完成狀態，並在儀表板上即時更新總體計畫進度。
[T4 Transparent 透明]： AI 生成的改善建議會標示其推理依據（例如：基於 GRI 305 標準、參考同業最佳實踐），並提供「查看 AI 評分邏輯」選項，解釋評分權重與考量因素。
[T5 Trustworthy 信任]： 健檢評分引擎採用經過第三方驗證的 ESG 評級方法論（如 SASB, TCFD），並定期更新，確保評估結果的專業性與可信度。所有輸入數據在計算前會進行數據完整性與一致性檢查。

5. 功能項目解說和使用技術 (Features & Tech Stack)
ESG 議題自動評估： 根據預設的 15 項核心 ESG 議題，結合企業輸入數據與內建評分模型進行自動化評分。技術使用 Python 的 Pandas 進行數據處理與計算，搭配 Scikit-learn 實現基於規則的評分引擎。
互動式儀表板渲染： 呈現總體得分、雷達圖、熱力圖、趨勢圖等。技術使用 Recharts 或 ECharts 進行圖表渲染，結合 React Context API 進行數據狀態管理，確保圖表與數據的即時同步。
AI 驅動改善建議： 根據企業在各議題上的得分與行業基準，利用 AI 生成具體、可執行的改善建議。技術使用 Google Gemini Pro 或 OpenAI GPT-4 進行自然語言生成 (NLG)，並結合內部知識庫進行微調 (Fine-tuning)。
90 天行動計畫追蹤： 提供計畫創建、任務分派、進度更新與提醒功能。技術使用 Supabase 的 PostgreSQL 搭配 Row Level Security (RLS) 進行任務數據管理，前端使用 React Hook Form 處理表單輸入，並結合 React Query 進行數據同步。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 當雷達圖數據點過多或標籤文字過長時，圖表必須自動調整字體大小或提供 Tooltip 顯示完整內容，不可出現文字重疊或溢出圖表邊界的情況。
🚨 邏輯/體驗紅線： 健檢評分結果與歷史數據相比，若出現超過 20% 的異常波動（例如：在無重大事件發生下，某議題得分突然暴跌），系統必須觸發警示，並提示使用者檢查數據來源或評分模型，避免誤導性分析。若 AI 建議與企業實際業務情境嚴重脫節，必須提供「回報不相關建議」的按鈕，並將該反饋納入 AI 模型優化。

7. 矩陣關聯 (Matrix Connection)
上游數據： 來自全系統數據倉儲 (`esg_data_warehouse`)。
下游影響： 生成行動計畫影響 `/tasks` 與 `/dashboard` 的風險預警指標。
依賴組件： BrandScoreCard, RadarChart, BrandProgressGauge, AIInsightCard.
