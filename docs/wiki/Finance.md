# ESG 財務績效分析 ESG Financial Performance Analysis [Finance]
路徑： /finance | 權限： CFO, Finance_Manager, CSO | 所屬旅程： VI. 績效評估與報告

1. 模組定位 (Core Purpose)
ESG 財務績效分析模組，旨在量化企業 ESG 投入的財務回報 (ROI)，並提供符合 TCFD 等國際標準的氣候相關財務揭露工具，協助企業將永續策略轉化為可衡量的商業價值。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 財務長 (CFO) 或財務經理。他們面臨的痛點是難以將抽象的 ESG 投資具體化為財務回報，且在準備 TCFD 等氣候相關財務揭露報告時，需耗費大量時間手動收集、整理與計算數據，導致報告效率低下且數據可信度存疑。
體驗高光時刻 (Aha Moment)： 當 CFO 在董事會上，點擊「綠色能源專案」的 ROI 卡片，系統即時彈出該專案的「五年財務效益預估圖」與「實際節省成本明細」，並自動生成符合 TCFD 框架的「氣候風險財務影響報告草稿」。CFO 瞬間感受到「永續不再是成本，而是可量化的投資回報」的巨大價值。
操作軌跡：
1. 財務長進入「ESG 財務績效分析」頁面，選擇欲評估的 ESG 專案（例如：再生能源導入計畫）。
2. 在專案配置區輸入初始投資成本、預期節省金額與時間週期。
3. 系統即時運算並在儀表板上顯示該專案的預期 ROI、IRR 等財務指標。
4. 點擊「生成 TCFD 報告」按鈕，系統自動整合相關數據，產出初步報告框架。
5. 檢閱並微調報告內容，最終匯出為 PDF 或 Excel。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 採用「儀表板型佈局」。頂部為核心 KPI 總覽卡片 (如：總 ESG ROI, 氣候風險曝險值)，下方為左右雙欄，左側為「專案列表與篩選器」，右側為「互動式圖表與數據表格」。
核心液態玻璃元件： BrandKPIWidget (核心指標卡片，支援趨勢圖顯示)、FinanceChart (多維度互動式圖表，如折線圖、柱狀圖、瀑布圖)、TCFDReportPreview (報告預覽區)。
行動端適配 (RWD)： < 768px 時，所有圖表自動收合為「可展開的卡片式列表」，僅顯示核心 KPI 數值與趨勢箭頭。點擊卡片可展開查看詳細圖表。專案列表轉為單欄垂直滾動，篩選器收合至頂部懸浮按鈕。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向： 財務數據（如：營收、成本、資本支出）從企業 ERP/會計系統（透過 API 或 CSV 匯入）流入，與 ESG 專案數據（來自 `project_management` 表）結合，運算結果存入 `esg_financial_impacts` 表，並影響 `dashboard_kpis`。
5T 實踐點：
[T1 Tangible 具體]： 將抽象的 ESG 專案效益（如：碳排放減少量）透過預設的碳權價格或節能成本模型，具體換算為可量化的財務節省金額與投資回報率 (ROI)。
[T2 Traceable 追溯]： 每一筆財務影響數據（例如：節省電費）都可追溯至其原始數據來源（如：電費單、能源監控系統數據）及相關的 ESG 專案。
[T3 Trackable 追蹤]： 任何對財務模型參數或 TCFD 報告內容的修改，系統都會自動記錄操作者、時間與變更內容，形成完整的審計軌跡。
[T4 Transparent 透明]： 系統會清楚揭露所有財務指標（如 ROI, IRR）的計算公式與所採用的假設參數（如：折現率、碳權價格），確保計算過程公開透明。
[T5 Trustworthy 信任]： 匯入的財務數據會經過自動化的數據完整性檢查與交叉比對，確保數據的準確性與可信度，防止人為錯誤或惡意篡改。

5. 功能項目解說和使用技術 (Features & Tech Stack)
ESG 專案財務效益計算引擎： 根據用戶輸入的專案成本、預期效益與時間週期，即時計算 ROI、IRR、NPV 等財務指標。技術使用 GoLang 後端服務搭配 `gonum/optimize` 庫進行複雜財務模型運算。
TCFD 氣候相關財務揭露報告生成器： 根據預設的 TCFD 框架，自動整合氣候風險、機會與財務影響數據，生成可編輯的報告草稿。技術使用 `Puppeteer` 或 `wkhtmltopdf` 進行 PDF 報告生成，前端使用 `React-PDF` 進行預覽。
互動式 ESG 財務儀表板： 提供多維度的數據篩選與鑽取功能，視覺化展示 ESG 投資組合的財務表現。技術使用 `ECharts` 或 `Plotly.js` 進行圖表渲染，結合 `React Query` 實現數據即時刷新。
數據整合與清洗： 支援從多個外部 ERP/會計系統（如 SAP, Oracle Financials）透過 API 或 SFTP 匯入財務數據，並進行自動化清洗與格式轉換。技術使用 `Apache Nifi` 或 `Airflow` 進行數據管道管理。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 當圖表數據點過多時，X/Y 軸標籤必須自動調整間距或採用「縮放/平移 (Zoom/Pan)」功能，不可出現標籤重疊或文字溢出圖表邊界的情況。
🚨 邏輯/體驗紅線：
*   ESG 專案的 ROI 計算結果與手動驗證的結果誤差必須小於 0.1%，若超過此閾值，系統必須亮起警示。
*   TCFD 報告生成失敗時，必須提供明確的錯誤訊息與解決方案，而非僅顯示空白頁面或通用錯誤碼。
*   當匯入的財務數據與系統內已存在的數據發生衝突或不一致時，必須彈出「數據衝突解決」對話框，引導用戶進行手動確認或選擇覆蓋策略，不可靜默處理導致數據混亂。

7. 矩陣關聯 (Matrix Connection)
上游數據： 來自企業 ERP/會計系統與 `/environmental` (能耗與碳排數據)。
下游影響： 提供氣候風險財務影響數據給 `/dashboard` 與 `/publish` (TCFD 揭露)。
依賴組件： BrandKPIWidget, FinanceChart, TCFDReportPreview.
