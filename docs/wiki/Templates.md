專家模板 Expert Templates
路徑： /templates | 權限： PM, ADMIN | 所屬旅程： II. 策略盤點與分派

1. 模組定位 (Core Purpose)
專家模板提供預建的國際 ESG 報告框架（如 GRI 2021, ISSB S1/S2）與零算力啟動模板，協助企業快速套用標準化結構，確保報告合規性與數據收集效率。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 永續專案經理 (PM) 或 ESG 顧問。他們在啟動新的永續報告專案時，常面臨「不知從何開始建立報告結構」、「如何確保符合最新國際標準」、「多個框架之間欄位如何對應」等痛點，導致初期設定耗時且容易出錯。
體驗高光時刻 (Aha Moment)： 當 PM 點擊「GRI 2021 模板」並在 3 秒內看到一個完整、預設好所有章節與數據欄位的報告結構，且每個欄位旁都標註了對應的 GRI 指標說明時，他們會感到「哇！原來建立合規報告可以這麼簡單、這麼有信心」。
操作軌跡：
1. 新任 PM 首次登入 ESG GO，被引導至模板頁面。
2. 瀏覽不同國際框架模板，點擊「GRI 2021」進行預覽。
3. 確認模板結構符合需求後，點擊「套用至新報告」。
4. 系統自動生成報告草稿，PM 隨即開始填寫內容，無需從零開始。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 採用「模板畫廊 (Template Gallery)」佈局。頂部為篩選與搜尋區，下方為多個 `BrandTemplateCard`，每個卡片展示模板名稱、版本、適用框架與簡要說明。點擊卡片後彈出 `BrandModal` 進行詳細預覽與套用操作。
核心液態玻璃元件： `BrandTemplateCard` (帶有框架標籤與版本號)、`BrandTag` (用於標示 GRI/ISSB)、`BrandModal` (用於模板預覽與確認)。
行動端適配 (RWD)： < 768px 時，模板卡片自動從多欄佈局轉為「單欄堆疊式卡片 (Stacked Cards)」，確保在小螢幕上內容清晰可讀。模板預覽 Modal 轉為全螢幕底部抽屜 (Bottom Sheet)，提供更大的觸控區域與沉浸式預覽體驗，避免內容被鍵盤遮擋。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向： 模板定義（JSON/YAML 格式）儲存於 `template_library` 資料表。當使用者套用模板時，系統會根據選定的模板，在 `report_structures` 或 `data_collection_schemas` 資料表中生成對應的報告章節與數據欄位。這些操作會觸發 `audit_logs` 記錄。
5T 實踐點：
[T1 Tangible 具體]： 將抽象的 GRI/ISSB 國際標準，具體化為可直接套用、可編輯的報告章節與數據收集欄位，讓使用者能直觀理解並操作。
[T2 Traceable 追溯]： 每次模板的套用操作，都會在 `audit_logs` 中記錄「使用者 ID」、「套用模板 ID」、「套用時間」與「目標報告 ID」，確保報告結構來源可追溯。
[T3 Trackable 追蹤]： 模板定義的每次更新（由誰、何時、修改了哪些欄位）都會被記錄在 `template_version_history` 中，確保模板內容的演進與變更歷史可追溯，便於審計與合規性檢查。
[T4 Transparent 透明]： 模板預覽時，每個欄位都會清晰標示其所屬的國際框架指標（例如：GRI 305-1），並提供官方指引的連結，讓使用者明確了解其合規性依據與定義。
[T5 Trustworthy 信任]： 平台內建的國際框架模板，會定期由 ESG 專家團隊進行審核與更新，並標註「官方認證」與「最新版本」標籤，確保其權威性與準確性。任何非官方或自定義模板會明確區分，避免混淆。

5. 功能項目解說和使用技術 (Features & Tech Stack)
模板畫廊與篩選： 支援依框架、版本、關鍵字進行模板搜尋與篩選，提供快速定位功能。技術使用 React `useState` / `useReducer` 進行本地狀態管理，搭配 `Fuse.js` 進行高效的模糊搜尋，並透過 `React Query` 進行遠端資料快取。
模板預覽與欄位映射展示： 點擊模板卡片後，以 Modal 形式展示模板的完整結構樹與各欄位對應的國際標準，支援展開/收合功能。技術使用 `react-json-view` 或客製化樹狀結構組件，並透過 `react-diff-viewer` 顯示不同版本間的差異，提升透明度。
零算力啟動嚮導 (Guided Onboarding)： 針對新用戶提供一步步的模板選擇與報告建立引導流程，降低上手門檻。技術使用 `react-joyride` 或客製化 `BrandWizard` 組件，結合 `localStorage` 記錄用戶進度，並透過 `PostHog` 追蹤用戶在嚮導中的行為路徑。
模板版本管理： 每個模板都具備版本號，並可查看歷史版本與其變更日誌。技術使用 Supabase 的 `versioning` 功能或在 `template_library` 表中維護 `version_id` 與 `is_active` 欄位，並結合 `Git-like` 的差異比較演算法來展示版本間的具體修改。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 當模板名稱或簡要說明過長時，`BrandTemplateCard` 上的文字必須觸發 `text-overflow: ellipsis`，並在懸浮時顯示完整 Tooltip，絕對不可撐破卡片容器或導致佈局混亂。在行動端，卡片高度應保持一致，避免因內容長度不一導致視覺跳動。
🚨 邏輯/體驗紅線：
    *   點擊「套用模板」後，若目標報告已存在內容，必須彈出二次確認對話框，明確告知「套用新模板可能覆蓋或重置現有結構」，並提供「保留現有內容並合併」或「完全覆蓋」的選項，防止資料意外遺失。
    *   若使用者嘗試套用已過期或被標記為「不建議使用」的模板，系統必須給予顯眼的視覺警告（例如：紅色邊框、警告圖示），並建議使用最新版本，同時提供「仍要使用」的選項，但需再次確認。
    *   在網路不穩定或斷線情況下，模板預覽應能顯示快取內容（若有），並在套用模板時給予明確的離線提示，避免使用者誤以為操作成功。
