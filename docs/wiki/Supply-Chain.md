**供應鏈透明 Supply Chain Transparency**
路徑： /supply-chain | 權限： Procurement Manager, Sustainability Manager, CSO | 所屬旅程： III. 供應鏈治理與風險

1. 模組定位 (Core Purpose)
供應鏈透明模組是 ESG GO 的核心治理中樞，整合供應商 ESG 評估、承諾追蹤與風險分級，提供企業對供應鏈進行全面透明化治理與風險管理的單一視圖。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 採購經理 (Procurement Manager) 或永續專員。他們需要管理數百甚至數千家供應商，手動追蹤每家供應商的 ESG 表現、風險等級和合規文件（如承諾書），耗時費力且容易出錯，難以快速應對法規要求或突發事件，導致供應鏈風險不透明，承諾文件與評分紀錄難以整合。
體驗高光時刻 (Aha Moment)： 當採購經理在儀表板上看到某高風險供應商的 ESG 評分突然下降，系統自動觸發警報並建議替代方案，同時顯示該供應商未簽署最新版承諾書。這種即時、整合的風險洞察，讓經理能即時採取行動，避免潛在的供應中斷或品牌聲譽受損，感受到「一切盡在掌握」的安心感。
操作軌跡：
1. 採購經理登入系統，進入「供應鏈透明」頁面。
2. 透過頂部篩選器，快速篩選出「高風險」且「ESG 評分低於 B 等」的供應商。
3. 點擊特定供應商，右側抽屜或彈出視窗顯示其詳細的 ESG 評估報告、歷史評分趨勢與未簽署的承諾書清單。
4. 系統自動建議發送「ESG 改善通知」或「承諾書補簽提醒」給該供應商。
5. 追蹤供應商的改善進度，並在供應商完成後更新其狀態，確保供應鏈合規。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 採用「頂部篩選區 + 下方資料表格」的經典佈局。頂部為多維度篩選與搜尋欄，下方 80% 區域為可排序、可篩選的「供應商列表 (Data Table)」。點擊供應商後，右側將滑出「詳細資訊抽屜 (Side Drawer)」顯示供應商詳細資料。
核心液態玻璃元件： BrandDataTable (具備多層篩選、排序與分頁功能), RiskBadge (顯示風險等級，如：高/中/低，並以紅/黃/綠顏色區分), ESGScoreGauge (顯示 ESG 評分進度條或環形圖，直觀呈現表現), CommitmentStatusToggle (承諾書簽署狀態開關或標籤，顯示已簽署/未簽署/待更新)。
行動端適配 (RWD)： < 768px 時，供應商列表自動轉化為「堆疊式資料卡片 (Stacked Cards)」，每張卡片顯示核心資訊（供應商名稱、ESG 評分、風險等級）。複雜的篩選器收合至頂部或底部浮動按鈕 (FAB)，點擊後以全螢幕模式展開。詳細資訊頁面則以全螢幕模式呈現，確保內容可讀性與操作性。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向： 供應商基本資料、ESG 評分、風險等級、承諾書狀態等數據，從 `supplier_profiles`、`esg_assessments`、`risk_matrix`、`commitment_documents` 等資料表撈取並整合。任何評分或狀態變更，會即時更新至對應資料表，並可能觸發 Dashboard 上的供應鏈風險 KPI 變動，確保數據的即時性與一致性。
5T 實踐點：
[T1 Tangible 具體]： 將抽象的供應商 ESG 表現，具體化為可量化的 ESG 評分 (A-F 等級或 0-100 分) 與風險等級 (高/中/低)，並以視覺化方式呈現，讓管理者能快速理解與決策。
[T2 Traceable 追溯]： 每次供應商 ESG 評分或風險等級變更，系統自動記錄變更者、變更時間與變更前後的數值，形成完整的 Audit Log，確保所有操作皆可追溯。
[T3 Trackable 追蹤]： 承諾書的簽署狀態、到期日、版本號等資訊，皆可被追蹤與管理，並可設定自動提醒功能，確保合規性。
[T4 Transparent 透明]： 供應商的 ESG 評分來源（例如：第三方評級機構數據、內部問卷結果）與計算邏輯，應在詳細資訊頁面中清晰揭露，增加數據的透明度與可信度。
[T5 Trustworthy 信任]： 供應商上傳的承諾書或其他佐證文件，應透過 Hash 驗證機制確保其原始性與未被竄改，防止文件造假，建立數據信任基礎。

5. 功能項目解說和使用技術 (Features & Tech Stack)
供應商列表與多維度篩選： 顯示所有供應商，支援多維度篩選（如：ESG 評分、風險等級、產業別、在地採購狀態）與關鍵字搜尋，提供快速定位功能。技術使用 React Table 或 Ag-Grid 進行高效能資料渲染，搭配 `react-query` 進行資料快取與伺服器狀態管理，確保數據載入與篩選的流暢性。
ESG 評分與風險分級視覺化： 以直觀的視覺化方式呈現供應商的 ESG 評分（如：環形圖、進度條）與風險等級（如：顏色標籤），幫助用戶快速理解。技術使用 Recharts 或 Chart.js 進行圖表渲染，結合 `react-spring` 或 `framer-motion` 實現平滑動畫，提升用戶體驗。
承諾書追蹤與文件管理： 顯示供應商是否簽署特定承諾書，並提供上傳、下載、預覽與版本管理功能。技術使用 Supabase Storage 或 AWS S3 進行文件儲存，前端使用 `react-dropzone` 進行文件拖曳上傳，並透過 `Web Crypto API` 進行文件 Hash 驗證，確保文件安全與完整性。
在地採購狀態標記與統計： 標記供應商是否符合在地採購標準，並可設定相關閾值，提供在地採購比例的統計分析。技術使用簡單的布林值開關或下拉選單，後端邏輯判斷供應商註冊地址與企業營運地點的地理位置資訊，並結合數據聚合計算，可利用 PostGIS 進行地理空間查詢。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 當供應商數量超過 1000 筆時，列表滾動必須保持流暢，不可出現卡頓或渲染延遲，確保大規模數據下的性能。篩選條件過多時，篩選器區域不可溢出或遮擋主要內容，應妥善處理為可滾動或收合的組件，避免佈局混亂。在行動端，卡片式佈局下，供應商名稱或關鍵資訊過長時必須觸發 `text-overflow: ellipsis` (顯示...)，配合 Tooltip 檢視完整內容，絕對不可撐破卡片容器。
🚨 邏輯/體驗紅線： 當篩選條件導致無任何供應商符合時，必須顯示友善的「無資料」提示，而非空白頁面或錯誤訊息，避免用戶困惑。任何對供應商資料的修改（如：更新評分、簽署承諾書），必須在成功後即時反映在頁面上，並給予視覺回饋（如：Toast 提示），確保數據同步與用戶感知。若系統檢測到供應商的 ESG 評分或風險等級發生重大變化（例如：從低風險變為高風險），必須自動觸發內部通知機制（如：郵件、站內信）給相關負責人，以便及時介入處理。
