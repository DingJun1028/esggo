控制台 Dashboard Dashboard
路徑： / | 權限： ALL_USERS | 所屬旅程： I. 平台總覽與決策

1. 模組定位 (Core Purpose)
作為 ESG GO 平台的統一入口與決策中樞，控制台透過整合關鍵績效指標 (KPI)、GRI/ISSB 覆蓋率、5T 活動日誌與治理總覽，提供企業高層與專案團隊一站式、即時的永續表現洞察。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 永續長 (CSO) 或永續專案經理 (PM)。他們每天面對海量的 ESG 數據與報告進度，難以在短時間內掌握整體營運的永續表現、報告撰寫進度以及關鍵治理活動，導致決策效率低落且錯失風險預警。
體驗高光時刻 (Aha Moment)： 當 CSO 登入系統，一眼掃過 Dashboard，發現「碳排放強度」的趨勢圖自動標示出異常峰值，並在旁邊的「5T 活動日誌」中看到相關部門已啟動調查的紀錄。CSO 瞬間感受到「系統主動預警並提供行動脈絡」的安心與高效。
操作軌跡：
1. 登入系統，Dashboard 自動載入，快速瀏覽各項核心 KPI (如碳排放、水資源消耗) 的即時數據與趨勢。
2. 點擊「GRI 覆蓋率矩陣」中的紅色區塊，直接跳轉至 SustainWrite 編輯器，補齊缺失的報告章節。
3. 檢閱「5T 活動日誌」，追溯近期關鍵數據的變更與審核紀錄。
4. 透過「快速導航卡片」點擊進入「重大性矩陣」或「證據金庫」，進行更深入的策略規劃或資料查核。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 採用「彈性 Bento Grid 佈局」，預設為 4 欄網格，根據內容重要性與資訊密度，動態調整卡片大小 (例如：KPI 儀表板佔 2x2 區塊，GRI 覆蓋率佔 1x2 區塊)。
核心液態玻璃元件： BrandMetricCard (顯示單一 KPI 數值與趨勢箭頭)、BrandProgressRing (GRI 覆蓋率圓環進度條)、BrandActivityFeed (5T 活動日誌列表)、BrandQuickNavCard (主要功能入口導航卡片)。所有卡片均採用 Liquid Glass 效果。
行動端適配 (RWD)： < 768px 時，Bento Grid 自動堆疊為單欄佈局，並優先顯示「核心 KPI」與「GRI 覆蓋率」卡片。5T 活動日誌預設收合，點擊後展開。快速導航卡片轉為底部 Tab 或可滑動的水平列表。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向：
*   KPI 數據：從 `kpi_metrics` 表與 `data_points` 表聚合計算，每 5 分鐘更新一次。
*   GRI 覆蓋率：實時查詢 `report_drafts` 表的 `completion_status` 字段。
*   5T 活動日誌：從 `audit_logs` 表中撈取最近 20 條記錄。
*   所有數據的變動都會影響 Dashboard 的即時顯示。
5T 實踐點：
*   [T1 Tangible 具體]：將抽象的永續目標（如碳中和）具體化為可量化的 KPI 數值、趨勢圖與進度條，讓管理者一眼掌握實際進展。
*   [T3 Trackable 追蹤]：5T 活動日誌清晰記錄所有關鍵數據的變更、審核與發佈事件，包含操作者、時間戳與變更內容摘要，確保所有治理行為可追溯。
*   [T5 Trustworthy 信任]：所有顯示的 KPI 數據，其來源都可點擊追溯至原始數據點或證據金庫中的佐證文件，建立數據的信任鏈。
*   [T0 Timely 即時]：Dashboard 數據每 5 分鐘自動刷新，確保使用者看到的資訊是最新的，避免因數據延遲導致的誤判。

5. 功能項目解說和使用技術 (Features & Tech Stack)
KPI 儀表板： 顯示關鍵 ESG 指標的即時數值、趨勢圖與目標達成進度。技術使用 Recharts 或 ApexCharts 進行圖表渲染，結合 React Query 或 SWR 進行數據緩存與實時更新。
GRI 覆蓋率矩陣： 以視覺化矩陣形式展示各 GRI/ISSB 章節的撰寫進度與完整度。技術使用 React Grid Layout 實現彈性佈局，並透過 Zustand 或 Recoil 進行全局狀態管理，實時反映 SustainWrite 的編輯進度。
5T 活動日誌： 顯示平台內所有與 5T 協定相關的數據變更、審核與發佈事件流。技術使用 Supabase Realtime 訂閱 `audit_logs` 表的變動，並以 Server-Sent Events (SSE) 推送到前端，實現近乎實時的活動更新。
主要功能入口導航： 提供快速跳轉至其他核心功能頁面的卡片式導航。技術使用 Next.js 的 Link 元件進行客戶端路由，並利用 Framer Motion 實現卡片懸停與點擊的微互動動畫。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 當 KPI 數值過大（例如：億級碳排放量）時，數字必須自動縮放或使用單位轉換（如 K, M, B），不可溢出 BrandMetricCard 容器。GRI 覆蓋率矩陣在不同螢幕尺寸下，卡片間距與文字大小必須保持視覺一致性，不可出現重疊或過度擠壓。5T 活動日誌的日期時間戳與事件描述，在行動端必須正確換行，不可導致水平滾動條出現。
🚨 邏輯/體驗紅線： 任何 KPI 數據若因後端服務異常導致無法載入，必須顯示友善的「數據載入失敗」提示，並提供重試按鈕，而非顯示空白或錯誤碼。5T 活動日誌的數據刷新必須平滑無感，不可出現頁面閃爍或數據突然跳動的情況。當使用者點擊 GRI 覆蓋率矩陣中的連結跳轉至 SustainWrite 編輯器時，必須確保編輯器能正確載入對應章節，且無任何延遲感。
