# ESG Academy ESG Academy [Academy]
路徑： /academy | 權限： ALL_USERS | 所屬旅程： I. 學習與能力建構

1. 模組定位 (Core Purpose)
ESG Academy 是 ESG GO 平台的核心學習與能力建構中心，提供系統化的 ESG 知識課程、實務案例與認證，旨在提升用戶的 ESG 專業素養，並將理論知識轉化為企業永續發展的實際行動力。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 企業的永續專案經理、高階主管、各部門填報窗口，以及對 ESG 議題感興趣的員工。他們常面臨 ESG 知識碎片化、法規更新快速、缺乏系統性學習路徑，以及難以將抽象概念應用於實際工作中的困境。
體驗高光時刻 (Aha Moment)： 當用戶在 Academy 中找到與自身產業高度相關的「GRI 305 排放量計算實務」課程，並透過互動式模擬工具，成功計算出企業的碳排放量，感受到「原來複雜的 ESG 數據，透過系統學習可以變得如此清晰且可操作」的成就感。
操作軌跡：
1. 新用戶首次登入，在儀表板看到推薦的「ESG 基礎入門」系列課程。
2. 點擊進入課程頁面，觀看由 Berkeley Haas 教授主講的影片模組。
3. 完成每章節後的互動式測驗，即時檢視學習成效。
4. 瀏覽「案例研究」區塊，下載與公司業務相關的永續報告範本。
5. 成功完成所有課程並通過最終測驗，獲得 ESG GO 頒發的電子證書。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 採用「左側課程導航樹 + 右側內容播放/閱讀區」的標準學習平台佈局。左側導航樹可收合展開，右側內容區根據模組類型（影片、文件、測驗）動態切換顯示。
核心液態玻璃元件： BrandVideoPlayer (自適應影片播放器)、BrandProgressTracker (學習進度條，帶有 Liquid 漸變效果)、BrandQuizCard (互動式測驗卡片，具備即時反饋)。
行動端適配 (RWD)： < 768px 時，左側課程導航樹自動收合為頂部「課程目錄」按鈕或底部 Tab。內容區（影片、文件、測驗）必須 100% 滿版顯示，影片播放器自動適應螢幕比例，測驗卡片堆疊顯示，確保觸控操作流暢。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向： 用戶的學習進度、測驗成績、證書狀態等數據，會即時寫入 `user_learning_progress` 和 `course_completion` 資料表。課程內容（影片 URL、文件路徑、測驗題目、合作夥伴資訊）則從 `academy_courses` 和 `academy_modules` 資料表讀取。
5T 實踐點：
[T1 Tangible 具體]： 將抽象的 ESG 知識點，透過影片、圖表、互動式測驗、案例研究等形式具體化，讓學習成果可量化（如：課程完成度百分比、測驗分數、證書）。
[T3 Trackable 追蹤]： 系統會精確記錄用戶在每個課程模組的學習時間、完成度、測驗嘗試次數與分數，並在後台提供詳細的學習報告，方便企業追蹤員工的 ESG 能力提升狀況。
[T4 Transparent 透明]： 課程內容的來源（如：與 Berkeley Haas、TSISDA 等知名機構的合作）會明確標示，確保知識的權威性與透明度，並提供參考文獻連結。

5. 功能項目解說和使用技術 (Features & Tech Stack)
課程管理與發佈： 後台提供直觀的課程編輯器，支援模組排序、內容上傳（影片、PDF、SCORM 包），並可設定課程發佈時間與權限。技術使用 Strapi CMS 或客製化後台管理系統 (基於 Next.js Admin Panel)。
互動式學習模組： 支援多種內容形式，包括高畫質影片播放、內嵌式 PDF 閱讀器、多選題/單選題測驗、拖曳配對題。技術使用 React Player (影片播放)、React-PDF (PDF 渲染)、客製化表單組件 (測驗，使用 React Hook Form)。
學習進度追蹤與證書頒發： 實時記錄用戶學習進度，並在完成所有課程與測驗後，自動生成帶有唯一驗證碼的可下載電子證書。技術使用 Supabase Edge Functions 處理證書生成邏輯，PDF.js 或 Puppeteer 生成 PDF 文件。
AI 學習助手： 提供基於課程內容的智能 Q&A 服務，用戶可隨時提問，AI 助手會從課程知識庫中提取相關資訊進行解答。技術使用 LangChain 結合 Gemini 2.0 Pro 模型，並將課程內容向量化儲存於 Supabase pgvector。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 影片播放器在不同解析度下必須保持正確的長寬比（例如 16:9），不可出現黑邊、畫面裁切或控制項被遮擋。課程導航樹在內容過多時必須有 Y 軸滾動條，不可溢出容器或導致頁面出現 X 軸滾動條。
🚨 邏輯/體驗紅線：
*   測驗提交後，若答案錯誤，必須提供清晰的錯誤提示和正確答案解析，而非僅顯示「錯誤」，以促進學習。
*   學習進度條必須即時更新，且在用戶跨裝置登入時能正確同步，不可出現進度回溯或不一致的情況。
*   證書生成必須在 5 秒內完成，且確保個人資訊（姓名、課程名稱、完成日期）正確無誤，證書上的驗證碼必須可被外部工具驗證。
*   在斷網情況下，已觀看影片的進度應能暫存於本地 (IndexedDB)，恢復連線後自動同步，不可彈出令使用者恐慌的錯誤訊息。
