在傳統的系統開發中，功能是被動等待使用者點擊的。但在 **ESGGO 善向永續** 平台中，我們打造了 **OmniMatrix (萬能元件心核｜終始矩陣)**。

## 2. 核心架構：六式循環 (The Six-Form Loop)
矩陣的核心運轉建立在精確的六個階段，每一個階段都環環相扣，形成閉環進化：

1. **覺式 (Awaken) - 感知邊界**
   - **職責**：接收外部混沌輸入，設定任務的 **定義完成標準 (DoD - Definition of Done)**。
   - **行為**：初始化 Context 與 TraceID，宣告系統進入運算狀態。

2. **解式 (Deconstruct) - 認知拆解**
   - **職責**：透過大語言模型 (Gemini) 進行第一層語義解析。
   - **行為**：將輸入分類為 `QUERY`, `MUTATE`, `REASONING`, `REPAIR`, `GENERATE` 等意圖，並抽取出具體行動步驟。

3. **策式 (Strategize) - 策略編排**
   - **職責**：決定如何最有效率地完成被拆解出的步驟。
   - **行為**：透過大腦決策，將任務精準派發給專精領域的 **子代理 (Sub-Agents)**，如 `Antigravity` (全端)、`Jules` (除錯)、`Pencil` (設計)。

4. **貫式 (Execute) - 降維打擊**
   - **職責**：真實跨平台執行任務。
   - **行為**：透過 OmniNexus (系統閘道) 呼叫對應代理的 MCP 工具或本地邏輯，落實 `actionGraph` 上的每一個節點。

5. **迴式 (Feedback) - 因果驗算**
   - **職責**：收斂執行結果並進行合規檢驗。
   - **行為**：與 `expectedDoD` 進行對齊，並透過 `OmniCore` 進行 **Hash Lock 密碼學封印**，確保該次運算的結果達到不可篡改的 5T 協議標準 (Truth, Trustworthy)。

6. **鍛式 (Forge) - 記憶沉澱**
   - **職責**：將這一次的成功經驗轉為系統的永久資產。
   - **行為**：寫入 `OmniMemory` 向量智庫中，確保下一次系統遇到類似的「因」時，能以更短的路徑直達「果」。
