# 審計日誌 Audit Log

**路徑：** `/audit-log`

## 功能定位
審計日誌頁是平台的治理稽核核心，記錄每一次資料、封印、驗證與操作事件。

## 功能說明
所有關鍵行為都會產生不可篡改的 5T 軌跡記錄，並附帶 SHA-256 哈希鎖定，用於日後追蹤與稽核。

## 主要內容
- 操作事件紀錄
- 操作者資訊
- 時間戳記
- 哈希鎖定
- 生命週期軌跡

## 使用方式
1. 進入審計日誌頁
2. 查詢指定事件或時間範圍
3. 查看操作人、內容與哈希資訊
4. 作為內外部稽核依據

## 使用場景
- 追查資料修改歷程
- 稽核與驗證流程
- 還原重要治理事件

## 頁面目標
- 建立操作可追蹤性
- 支援內部與外部查核

## 解決的痛點
- 系統缺乏可追溯操作紀錄
- 修改資料後難以重建歷史

## 目前成果
- 已建立 RECORDS 型態資料
- 已具備 hash 與稽核欄位邏輯
