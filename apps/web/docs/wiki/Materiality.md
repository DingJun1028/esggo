# 重大性矩陣 Materiality [Materiality]
路徑： /materiality | 權限： CSO, PM | 所屬旅程： II. 策略盤點與分派

1. 模組定位 (Core Purpose)
用於執行雙重重大性評估 (Double Materiality)。將抽象的 ESG 議題，透過財務衝擊與環境衝擊分數，轉換為可視化、可排序的象限氣泡圖，協助企業聚焦核心永續策略。

2. 客戶旅程與 UX 體驗 (Customer Journey & UX)
目標對象與痛點： 永續長 (CSO) 或永續專案經理 (PM)。他們需要根據 GRI 3-1 ~ 3-3 等標準，識別並評估企業面臨的重大 ESG 議題，但傳統方式耗時且難以即時調整與視覺化，導致會議決策效率低下。
體驗高光時刻 (Aha Moment)： 當 CSO 在跨部門策略會議上，直接拉動網頁上的分數滑桿，下方的「溫室氣體排放」氣泡立刻平滑飛入右上角的「核心重大象限」。這種即時算繪的科技感，極大提升了 CSO 在會議中的專業權威感與決策效率。
操作軌跡：
1. 匯入或選擇本年度待評估的 ESG 議題清單。
2. 統整各方評分後，在介面上透過拉動 Sliders 進行財務衝擊與環境衝擊的最終權重微調。
3. 觀察氣泡圖的動態落點，辨識出位於高重大性象限的核心議題。
4. 點擊「鎖定矩陣」，產出該年度不可篡改的策略基準與報告快照。

3. UI/UX 視覺與 RWD 基準 (Visual & RWD Baseline)
桌面版佈局： 頂部為分數拉桿列 (Sliders Array)，下方 70% 區域為滿版的互動式二維氣泡圖，清晰展示財務與環境衝擊的雙軸。
核心液態玻璃元件： BrandSlider (具備數值即時反饋與刻度標示)、MatrixBubble (帶有 Liquid 模糊邊緣的氣泡，可顯示議題名稱與分數 Tooltip)、BrandTooltip (具備智慧偏移邏輯)。
行動端適配 (RWD)： < 768px 時，預設顯示為「議題優先級列表 (List View)」，提供快速瀏覽與編輯功能。同時提供「點擊展開圖表」的橫向全螢幕選項，確保在行動裝置上也能流暢操作互動式氣泡圖，避免手指誤觸。

4. 核心邏輯與 5T 協定 (Logic & 5T Protocol)
資料流向： 使用者調整滑桿分數時，系統即時運算 ESG 議題在財務與環境衝擊軸上的權重與座標，並將變更寫入 `materiality_topics` 資料表，同時更新相關的 `audit_log`。最終鎖定後的矩陣快照會存入 `materiality_snapshots` 表。
5T 實踐點：
[T1 Tangible 具體]： 將 1-5 分的抽象問卷評分，具體化為二維座標軸上的絕對落點與氣泡大小，直觀呈現議題的重大性，讓抽象概念可視化。
[T2 Traceable 追溯]： 每次使用者調整滑桿分數，系統自動在該議題的 `audit_log` 中留下 `Score_Changed` 事件紀錄，包含操作者、時間與前後數值，確保決策過程可追溯。
[T3 Trackable 追蹤]： 鎖定矩陣後，系統會生成一個唯一的版本 ID，所有後續的報告或分析都可追溯到此特定版本的重大性評估結果。
[T4 Transparent 透明]： 氣泡圖的每個軸線定義、氣泡大小的計算邏輯，以及各議題的原始評分來源，都可在資訊提示中查看，確保評估方法的公開透明。
[T5 Trustworthy 信任]： 鎖定後的重大性矩陣，其資料表權限會被設定為 Read-Only，確保該年度的策略基準不可被任意篡改，提升數據的信任度。

5. 功能項目解說和使用技術 (Features & Tech Stack)
互動式氣泡圖渲染： 支援流暢的座標移動動畫、氣泡大小動態調整與 Tooltip 懸浮提示。技術使用 `Recharts` 搭配客製化 `SVG` 元素，或直接以 `D3.js` 實作力導向 (Force-directed) 防重疊邏輯，確保視覺清晰與效能。
雙重重大性狀態同步： 上方分數拉桿與下方氣泡圖需保持毫秒級同步，確保使用者操作的即時反饋。技術使用 `React useMemo` 計算權重，結合 `Zustand` 或 `Context API` 進行狀態共享。
矩陣鎖定與版本管理： 提供「鎖定矩陣」功能，將當前評估結果封存為不可修改的年度基準。技術使用 `Supabase` 的 `Row Level Security (RLS)` 變更資料表權限為 `Read-Only`，並搭配版本控制機制。
議題數據匯入/匯出： 支援從 Excel/CSV 匯入議題清單及評分數據，並可將最終矩陣結果匯出為 PDF 或圖片。技術使用 `PapaParse` 處理 CSV/Excel 匯入，`html2canvas` 或 `Puppeteer` 進行圖表匯出。

6. 品質達標與驗收紅線 (QA Red Lines)
🚨 UI 跑版紅線： 當兩個或多個議題分數極度接近時，氣泡重疊處的 `Tooltip` 必須具備「智慧偏移 (Smart Placement)」邏輯，不可出現互相遮擋字體或超出視窗邊界的問題。
🚨 邏輯/體驗紅線： 「鎖定矩陣」點擊後，必須彈出二次確認對話框，防止誤操作。鎖定後任何嘗試拉動滑桿的行為都應給予「已鎖定」的視覺阻尼回饋 (Visual Spring Feedback) 與禁用狀態，而非按鈕突然消失導致突兀感或無效操作。
🚨 資料一致性紅線： 在多使用者協作情境下，若多個使用者同時修改同一議題的分數，系統必須能正確處理衝突並提示使用者，避免資料覆蓋或不一致。
